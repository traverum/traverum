# 20-03-2026

## Goal
Perfect UI/UX for experience detail page: focused, not crowded, one thing at a time. Inspired by Expedia (map, what's included) but minimal; hybrid booking layout.

## Done

### Layout and flow
- **Hybrid booking:** Sticky sidebar = lightweight **PriceAnchor** (price + "Check availability" CTA that scrolls to `#booking`). Full booking flow moved to a **full-width BookingSection** below content (desktop). Mobile unchanged: bottom bar + BottomSheet.
- **Content order:** Image gallery → Title + category + meta (duration, capacity, languages) → **Tabs** (experience info) → Meeting point + Google Maps link → **Booking section** (with how-it-works + cancellation text).
- **Meeting point** kept separate (not in tabs); **Provided by** section removed.

### Tabs (experience info only)
- **DetailTabs** component: **Description** (default, first tab), **Highlights**, **What's included** (included / not included columns). Same pill + animated bubble style as category filter (`layoutId="detail-tab-bubble"`). No icons/emojis in tab labels.
- Description moved inside tabs (was above); default open tab = Description; tab order left-to-right: Description, Highlights, What's included.

### Booking section redesign
- **BookingSection** restructured: clear heading "Book this experience" + one-line booking explanation; **two-column grid** (3 cols calendar + sessions, 2 cols summary card). Right card: participants, price breakdown, total, CTA, cancellation policy below. No more floating blocks; cancellation text moved into the section (no separate two-column blurb above).

### New/updated components
- **PriceAnchor.tsx** — sidebar: display price + unit, "Check availability" button (smooth-scroll to `#booking`).
- **BookingSection.tsx** — full-width section: booking copy, grid with SessionPicker left and summary card right (participants, language when request, price, CTA, cancellation).
- **DetailTabs.tsx** — client tabs for Description, Highlights, What's included; Framer Motion bubble; description prop (rich text from page).
- **SessionPicker** — added `layout` prop: `responsive` (side-by-side in BookingSection) / `stacked` (e.g. narrow panels).

### Pages
- **Hotel detail** `[hotelSlug]/[experienceSlug]/page.tsx` and **Veyond direct** `experiences/[experienceSlug]/page.tsx`: same structure; category from `tags[0]`; placeholder data for highlights/included/not-included (always-on for now, no `?preview=placeholders` gate). Google Maps link from `location_address` or `meeting_point`.

### Branch and backup
- Branch **feature/experience-detail-page-restructure** created; all of the above committed; pushed to origin. PR link: https://github.com/traverum/traverum/pull/new/feature/experience-detail-page-restructure

## Decisions
- Hybrid layout: price anchor in sidebar, full booking below (not full panel in sidebar). Keeps main content readable; booking gets space.
- Tabs for experience content only (Description, Highlights, What's included); booking/cancellation live in the booking section.
- Description first in tabs and default open; order Description → Highlights → What's included.
- No "Provided by" on detail page.
- Placeholder highlights/included/not-included shown for all (no query param); can gate later when DB columns exist.
