## The problem (exact errors we saw)

1. **Wrong FK constraint names in DB**  
   The code uses PostgREST relationship hints like `hotel_configs_partner_fk`, `distributions_experience_fk`, `experiences_partner_fk`. The **actual** constraint names in the database are different (Postgres default naming), e.g. `hotel_configs_partner_id_fkey`, `distributions_experience_id_fkey`, `experiences_partner_id_fkey`.  
   Error from Supabase: *"Could not find a relationship between 'hotel_configs' and 'partners' in the schema cache"* using hint `hotel_configs_partner_fk`.

2. **Missing columns on `partners`**  
   The code selects `stripe_account_id` and `stripe_onboarding_complete` from `partners`. Those columns **do not exist** in the current database.  
   Error from Supabase: *"column partners_2.stripe_account_id does not exist"*.

So: **the database must be brought up to date** so that (1) the relationship names the code expects exist, and (2) `partners` has the Stripe columns the code selects. Until then, hotel pages will keep returning 404.
